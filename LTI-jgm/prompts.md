# Prompt principal

Eres un experto en análisis y diseño de sistemas de software. Tu misión es diseñar completamente un sistema ATS (Applicant-Tracking System) innovador para la startup LTI, que busca revolucionar el mercado de reclutamiento.
Información de la Empresa
LTI es una startup que quiere desarrollar el ATS del futuro, diferenciándose de la competencia mediante:
* Mayor eficiencia para departamentos de HR
* Colaboración en tiempo real entre reclutadores y managers
* Automatizaciones inteligentes
* Asistencia de IA en diversas tareas del proceso de reclutamiento

Entregables Requeridos
1. Descripción del Sistema y Modelo de Negocio
* Descripción breve del software LTI (2-3 párrafos)
* Valor añadido y ventajas competitivas específicas
* Funciones principales del sistema
* Diagrama Lean Canvas completo del modelo de negocio
2. Casos de Uso
* Identificar los 3 casos de uso principales del sistema
* Para cada caso de uso, proporcionar:
   * Descripción detallada
   * Actores involucrados
   * Flujo principal de eventos
   * Precondiciones y postcondiciones
   * Diagrama de casos de uso correspondiente
3. Modelo de Datos
* Entidades principales del sistema
* Atributos de cada entidad (nombre y tipo de dato)
* Relaciones entre entidades (cardinalidad incluida)
* Diagrama Entidad-Relación visual
4. Diseño de Alto Nivel
* Arquitectura del sistema explicada
* Componentes principales y sus responsabilidades
* Interacciones entre componentes
* Diagrama de arquitectura de alto nivel
5. Diagrama C4
* Seleccionar uno de los componentes del sistema
* Desarrollar un diagrama C4 completo que incluya:
   * Nivel 1: Contexto del sistema
   * Nivel 2: Contenedores
   * Nivel 3: Componentes (del componente seleccionado)
   * Nivel 4: Código (opcional, si es relevante)

Instrucciones Específicas
Para la Investigación:
* Considera las tendencias actuales en HR Tech
* Analiza las limitaciones de los ATS existentes
* Identifica oportunidades de diferenciación con IA y automatización
Para los Diagramas:
* Utiliza notación estándar (UML para casos de uso, Chen o crows foot para ER)
* Incluye leyendas cuando sea necesario
* Asegúrate de que los diagramas sean claros y profesionales
Para el Diseño Técnico:
* Considera escalabilidad y rendimiento
* Incluye aspectos de seguridad (datos sensibles de candidatos)
* Piensa en integraciones con otras herramientas (LinkedIn, plataformas de empleo)
* Considera la experiencia del usuario tanto para reclutadores como para candidatos

Formato de Respuesta
Organiza tu respuesta en secciones claras siguiendo el orden de los entregables. Para cada diagrama, proporciona primero la explicación textual y luego el diagrama correspondiente usando el formato más apropiado (Mermaid, ASCII art, o descripción detallada para posterior creación).
Criterios de Calidad
* Completitud: Todos los entregables deben estar presentes
* Coherencia: Los diferentes artefactos deben ser consistentes entre sí
* Innovación: El sistema debe mostrar claras ventajas competitivas
* Viabilidad técnica: Las soluciones propuestas deben ser realizables
* Orientación al usuario: Considerar tanto la experiencia del reclutador como del candidato

Vamos a empezar, comienza con tu análisis.
