# LTI ATS – Documentación de Diseño y Desarrollo

Este repositorio contiene la documentación completa del proceso de diseño y definición del sistema Applicant Tracking System (ATS) para la startup LTI. Aquí encontrarás tanto el proceso de trabajo colaborativo como los entregables de cada fase, organizados para facilitar su consulta y revisión.

## Índice de Fases y Artefactos

- [prompts.md](./prompts.md): **Documento principal**. Contiene todo el proceso de definición, brainstorming, prompts, respuestas y decisiones tomadas para cada fase. Es la referencia integral del proyecto.

- [contexto.md](./contexto.md): **Fase 1 – Investigación y Análisis**
  - Descripción del producto, análisis de competencia, propuesta de valor y Lean Canvas.

- [casos-uso.md](./casos-uso.md): **Fase 2 – Casos de Uso**
  - Descripción detallada de los principales casos de uso, diagramas y flujos.

- [entidades.md](./entidades.md): **Fase 3 – Modelado de Datos**
  - Entidades, atributos, relaciones y diagrama entidad-relación (ER).

- [arquitectura.md](./arquitectura.md): **Fase 4 – Diseño de Alto Nivel**
  - Arquitectura general, descripción de componentes, diagrama de arquitectura y consideraciones técnicas.

- [diagrama-c4.md](./diagrama-c4.md): **Fase 5 – Diagrama C4 (Nivel 3)**
  - Profundización en el módulo de IA para criba de CV, subcomponentes, interfaces y diagrama C4.

## ¿Por dónde empezar?

1. Lee primero `prompts.md` para entender el enfoque, decisiones y contexto general.
2. Consulta cada archivo de fase para ver el resultado final y los artefactos generados.
3. Revisa los diagramas en la carpeta `img/` para visualizar la arquitectura y los modelos.
